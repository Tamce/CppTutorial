＊提示：C＋＋语言是严格区分大小写的，但是忽略空白符（换行／制表符／空格），也就是说在不改变句意的情况下空格／回车／制表符是可以互换的且不限制数量，因此可以使用一些空格来排版使代码结构看起来更为清晰。另外，C＋＋语言的每一条语句都必须由分号“；”结束（观察源代码明确“语句”的概念）。

看起来，我们似乎成功编写出了一个Hello World程序，但是对于源代码中语句的作用并不了解，接下来我们来较为详细地介绍各语句作用。

＊[以下有两种教学方案，1是慢慢将每一个语句以及附属概念都大致解释清楚；2是根据我看得一本书上的教学方案，只关注简单的部分（能直接看得懂的部分），然后逐步深入（一些基本的概念会给出近似的解释而不是真正的解释）；希望大家能够给给建议，有需求的话我按方案二再写一条支路，这里就直接先按方案一走了]

看整个源代码的部分，我们发现第一行的代码似乎最奇怪，与其他代码格格不入，因为这一行代码带了一个‘＃’开头，而且在IDE中，这一整行代码都变成了同一种颜色（IDE中用不同颜色标记代码作用是为了区分关键字等，便于阅读）。这一行代码其实是一条预处理语句（带有＃开头的语句都是预处理语句），它严格来说不属于C++语言规范的一部分，因为它是用来叫编译器去做一些事情的，在前面我们说过，编译器在编译时会先对源代码做一个预处理，而这行代码会影响预处理的行为，我们看代码的内容： 

#include <iostream>

“include“是包含的意思，一对尖括号将这个文件名圈了起来并告知了这个文件的位置，因此这条代码是告诉编译器：”这个源代码要变成程序需要这个东西“，我们包含的是一个头文件[头文件类似于一个形容如何使用库的指引]，将别人的代码、已经实现的功能比作书本，这些书本组成了一个图书馆[Libarary库]，去寻找一本特定的书十分困难，所以我们需要一个索引，一个指南，这里的头文件就相当于这个指南，有了它，编译器在编译前按照这个指南把书都拿出来，就能够在这一小部分书中顺利地找到我们所需要的书本并和你自己所撰写的内容打包到一起组成一个完整的程序（对于这个过程如果不能很好理解，就简单理解为：要用到什么东西就要包含什么东西）。

那么，被我们所包含的'iostream'里面都有些什么东西呢？从文件名来理解，i-o-stream[in-out-stream]，意思是输入、输出、流，也就是说，标准库（由C＋＋标准化委员会提供的库[Libarary][已经完成的功能]）中掌管输入输出的东西都在这里面，那么“流”是什么意思呢？流是一种数据的传输方式，这里不再多说，有兴趣的可以上网查找相关资料。

另外，标准库全名为"标准模板库（Standard Template Libarary - STL）。

代码第二行： 

using namespace std;

为了更好说明这行代码的作用请先对代码做以下几种改动观察效果（做实验）：

1.删除本行并将源代码中的所有cout/cin改为std::cout/std::cin

2.删除本行，在此位置添加： 

using std::cout;
using std::cin;

对以上几种实验可以进行组合修改，或者故意不写某些内容观察现象，并根据英文"using"的意义猜测作用。以下是详细解释，建议自己先动手做以上实验并进行思考。

这一行的作用使用(using)了一个命名空间(namespace)，又一个新概念出现了！命名空间是为了防止命名重复而设计的一个东西，因为在编程时使用到其他人的东西是很常见的事情，但是不同人的代码中可能会有相同名称的东西，当存在重复名称的不同头文件被你用"include"同时包含的时候，就会出现名称不具有唯一性的问题，继续使用上面图书馆的例子，你用#include把甲乙两个图书馆的部分书籍（甲提供的库和乙提供的库）放到了一个房间里面，在源代码中发现你使用了语文书，在编译器试图把你自己的东西和你所用到的书（语文书）打包时，发现放到房间的一堆书中有两本语文书从而不能区分（一本来自甲图书馆，一本来自乙），于是接下来的过程就无法完成了。但是，如果在把图书馆的书放到房间时再用不同盒子装起来，标注它们的来源，这样就能正确区分两本语文书，因此甲图书馆决定搬书时都搬到名为"甲"的箱子里，而乙图书馆决定使用名为"乙"的箱子，但是这时候我们使用语文书的时候就必须说明"甲的语文书"或"乙的语文书"，如果甲语文书十分常用，我们就会希望提到语文书时默认为"甲的语文书"，但是编译器只能直接看到房间里的东西，所以要使用using语句来把某些书从盒子里拿出来，例如using std::cout（拿出std里的cout），而using namespace std则是将整个名为"std"的命名空间（盒子）中的所有东西都拿了出来，因此std::cout可以不用强调是std::（std里的），名为"std"的命名空间是标准库使用的唯一命名空间（意为标准），标准库的所有内容都在此命名空间中。

对于以上概念如果不能很好理解，那么在每个程序源代码的开头都加上"using namespace std;"语句即可。


从第三行开始，在第四行紧跟着的大括号的范围一起是一个“函数Function／方法”，“int main()"这一行被称为函数头，而由大括号括起的部分被称为函数体： 

int main()
{
}

这个函数的名字叫做"main"，意思是：主要的，因此也叫做主函数，主函数的作用是规定了程序的入口，一个程序的功能可能会很多，指令也会很多，但是一个程序要执行，就必须要有一个开始的地方，而main函数的作用就是指明了程序开始执行的位置，一个函数有时也被称为子程序，每一个函数都有它们自己的功能，每一个函数也都是有一定范围的，哪些过程哪些指令属于哪个函数都是十分明确的，所以每一个函数都有一对大括号来规定函数的范围。在计算机编程时的函数概念和数学中的函数概念十分相似，数学中的函数有一个自变量和一个因变量（输入该函数的信息和该函数返回的信息），但因为计算机处理的数据常常不仅限于数字，而且计算机中存储的数据均是以二进制的方式存储的，所以每一个信息都要有一个数据类型，这样计算机才能够正确处理各种信息，而在编程中使用到的函数也常常比数学中的函数复杂，数学中的函数常常只有一个自变量（输入），也只会得出一个结果（输出），而计算机所处理的信息常常十分复杂，所以这里的函数的输入可以有很多，输出也可以是一些更复杂的信息，当然由于上面所说的原因，一切数据都需要指明数据类型才能被正确处理，所以函数的输入和输出都需要标明数据类型。我们再次看到main函数的函数头：int main()，函数头包含了该函数的基本信息：“输入[参数]，输出[返回值]，函数名称（中括号内为术语）”，一个函数头看起来像是这样： 

返回值类型 函数名称 (参数列表)

返回值类型说明这个函数在经过执行之后得出的结果的类型，函数的返回值表明了函数在执行计算后得出的一个结果（结论／状态），这个结果会交给使用[调用Call]这个函数的对象（可以是其他的程序／其他的函数或者操作系统等），参数列表标明这个函数在执行时所需要的数据（可以不止有1个，因此叫参数列表，也可以为空）；当参数列表为空的时候括号中可以不写东西，也可以写“void"（空），函数也可以没有返回值，这时返回值类型不能省略，必须写“void“。

main函数的名称是“main“，这个名称不能改变，因为规定程序的入口位置就是main函数，这个程序中函数的参数是空，表示不接受任何数据就可以执行，返回值是“int”（表示整数）类型，ISO（International Standard Orgnization国际标准化组织）规定C＋＋ main函数必须返回一个整数型，main函数的返回值通常会交给操作系统（因为通常是由操作系统来打开这个程序，或者我们在IDE中设置了“程序执行完毕后自动暂停”的选项之后，停下来哦们所看到的“Return value"就是 main函数的返回值，主函数的返回值通常用于表明这个程序的执行状态（是正常执行完毕了还是发生错误退出了等等，一般为了方便使用也符合约定俗成的习惯，返回0表示程序正常运行完毕退出）。

说完函数我们直接看到代码的倒数第二行： 

return 0;

根据上面的说明以及对英文单词的理解，我们很快就能知道这条语句的作用就是“返回一个值，然后结束函数”，这里返回的是0，表示程序正常退出，当main函数结束之后，程序就退出了。有时候不写这样的一条语句也能够顺利编译和运行，而且程序运行起来没有什么区别，那是因为有些编译器自动在main函数执行完毕（执行到｝的时候）的时候返回了一个0，但这种写法是不标准的，有些编译器也会给出警告和错误信息，所以只要函数的返回值不是“void“，那么函数都必须使用return语句返回一个值。


到此，Hello World的程序就只有两条语句没有说明了，我们一条条来看：

首先是 

cout << "Hello World!";

cout是标准库提供的用于输出的一个东西[对象Object]，其意义为：”console-out控制台＊输出“，它在iostream中定义，所以一旦使用cout，就必须使用#include来包含iostream这个文件，而cout这个名字又是定义在名为std的命名空间中的，所以，如果没有“using namespace std;"这条语句，这里的cout就要改写为"std::cout"，cout后面是两个小于号，它们共同组成了一个符号“<<"，这个符号是运算符（数学中常用的运算符只有+-*/，而在编程时使用的运算符有很多，每一种运算符都有其自己的作用）的一种，用这个运算符把cout（负责输出的对象）和要输出的数据连接起来就可以达到输出的目的，我们输出的内容是"Hello World!"，之所以要加引号是为了说明这一块内容不属于起作用的源代码的一部分，而只是一个普通的字符串[文字符号构成的串string]数据。


＊控制台，我们现在所编写的程序都叫控制台应用程序，是没有花哨漂亮的用户界面的黑色窗口程序，由于以控制台为载体运行所以叫做控制台应用程序，虽然控制台应用程序与用户交流的方式很单一（大多数时候控制台应用程序都是和其他程序互相交流数据的），一个程序的核心并不在于如何与用户交流，而是其解决问题的思路，大部分的程序都是控制台应用程序，我们所说的”软件“通常由多个独立的程序组成，而负责与用户交流的带有用户界面的程序很少。大家可以打开任务管理器看看，任务列表里的类型如果是：”console“就说明其实控制台应用程序。 

cin.get();

cin和cout相同，其意义是“console-in“，是一个负责控制台的输入的对象＊，同样也在iostream中定义，存在于std的命名空间中。这行代码看起来似乎和之前所说的函数有相似的地方，因为它也有一对括号，没错，这里其实是一个函数调用，点号可以解释为“的”，所以这里是cin的get方法[函数]，这个函数的功能是获得用户输入的某些信息，这也就是为什么程序在停下来时按下回车才能结束，而其他按键看起来只是进行了一些输入。


C＋＋是面向对象的编程语言，尝试将计算机中各种抽象的数据、关系描述为现实世界中一样，因此对象[Object]是一个实体（比如说某一个人张三），这个对象拥有其自己的属性（比如说张三的体重），同时这个对象还可以做一些事情如cin.get，cin这个对象去get（张三去吃饭）。关于面向对象的概念以后再详细讲述，现在只需要学会如何使用好cout和cin即可。
